Os 3 principais modelos para usar

Como entender os preços

A Anthropic cobra por tokens — a unidade mínima de texto que o modelo processa. Na prática, 1 token equivale a cerca de 4 caracteres em inglês; em português, um pouco mais. Uma página A4 densa tem aproximadamente 600–700 tokens.

Os preços são sempre expressos por milhão de tokens — o que equivale a cerca de 750 mil palavras, ou algo como 1.500 páginas de texto. Na maioria dos usos do dia a dia, você está consumindo frações desse volume.

Há dois tipos de cobrança: input (o que você envia — seu prompt, documentos, histórico da conversa) e output (o que o modelo responde). O output custa entre 3× e 5× mais, dependendo do modelo.

Haiku 4.5
Input US$ 1 · Output US$ 5
Volume, velocidade, tarefas repetitivas.
  • Classificação e tagging de documentos
  • Resumos curtos e FAQs
  • Conversão de formatos (CSV → JSON)
  • E-mails padronizados
  • Pipelines de alto volume

Evitar: raciocínio multi-etapas, código complexo.

Padrão recomendado
Sonnet 4.6
Input US$ 3 · Output US$ 15
80–90% do trabalho. Melhor custo-benefício.
  • Escrever, revisar e refatorar código
  • Redação: posts, docs, relatórios, propostas
  • Síntese de múltiplas fontes
  • Análise de dados e workflows multi-step
  • Agentes com tool use

Evitar: raciocínio extremamente difícil ou agentes autônomos de horas.

Opus 4.7
Input US$ 5 · Output US$ 25
Quando qualidade é inegociável e Sonnet falhou.
  • Arquitetura de software complexo
  • Debug de race conditions e memory leaks
  • Análise jurídica / regulatória densa
  • Agentes autônomos de longa duração
  • Imagens de alta resolução (diagramas)

Evitar: uso reflexivo "para garantir". Se Sonnet serve, use Sonnet.

Novidades do Opus 4.7 — lançado em 16/04/2026

Tokenizer atualizado O mesmo prompt pode gerar até 35% mais tokens de input. Custo invisível se não monitorado.
Mais raciocínio interno O modelo "pensa mais" em conversas longas. Gera mais tokens de output — melhor qualidade, custo maior.
Recomendação prática Delegue a tarefa completa no primeiro turno: objetivo, escopo, restrições. Micro-ajustes multiplicam o consumo.

5 práticas para não "acabar rápido"

01
Escolha o modelo antes de começar. Não deixe tudo em Opus por padrão.
02
Conversas curtas e focadas. Inicie uma nova conversa para cada tarefa.
03
Envie só o necessário. Não cole PDFs inteiros para perguntar algo específico.
04
Peça concisão no prompt. "Responda em no máximo X frases." Output custa mais.
05
API: prompt caching + Batch API. Até 90% de redução no custo de input.
